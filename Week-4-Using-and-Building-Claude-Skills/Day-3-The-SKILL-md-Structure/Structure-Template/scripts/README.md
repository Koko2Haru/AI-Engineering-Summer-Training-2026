# scripts/

Deterministic logic that shouldn't be re-derived by reasoning each run — extraction, parsing, regex-based detection, fixed-formula computation. A script should know its own limits: if part of the task genuinely needs judgment a script can't provide, it should say so explicitly rather than returning a fabricated-precise answer.

Left empty in this template on purpose: Day 3's brief is the folder layout and a first-draft `SKILL.md`, not populated content. For a fully populated example of this folder in practice (3 real, tested scripts), see `../../../Day-4-Build-Your-Own-Skill/cv-reviewer/scripts/`.
