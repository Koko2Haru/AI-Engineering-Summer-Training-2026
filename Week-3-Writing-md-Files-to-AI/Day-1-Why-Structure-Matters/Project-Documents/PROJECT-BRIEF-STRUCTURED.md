# Coddiction — Structrued Project Brief


## 1. Overview

**Coddiction** (*code* + *addiction*) is a single-page, browser-only mini competitive
programming site. The player receives a session of AI-generated coding problems, solves
them in a built-in editor against a countdown timer, and gets an instant verdict for
each submission. When the session ends, a results screen shows the final score.

**One-line pitch:** LeetCode, shrunk to one HTML file, gamified to be addictive.

## 2. Goals

- Make solving coding problems feel like a game session: timer pressure, score, verdicts, ending screen.
- Run entirely in the browser — no server, no accounts, no installation.
- Use AI to generate the problems, including how many and how hard.

## 3. Core Features (v1.0)

| # | Feature | Description |
|---|---------|-------------|
| 1 | Problem view | Displays the current problem: title, difficulty, statement, function signature, visible examples |
| 2 | Code editor | A monospace editor where the player writes a JavaScript `solve()` function |
| 3 | Run | Executes the code against **visible** test cases only; shows output or the error message |
| 4 | Submit | Executes against **all** test cases (visible + hidden) and returns a verdict |
| 5 | Verdicts | `Accepted` / `Wrong Answer` (expected vs actual shown) / `Runtime Error` / `Time Limit Exceeded` |
| 6 | Session timer | A countdown for the whole session; visible at all times |
| 7 | Score | Points awarded per solved problem, scaled by difficulty |
| 8 | AI-generated problems | Problem set (count + difficulty mix) is generated by an AI in the site's problem JSON format |
| 9 | Ending screen | Shown when the timer expires or all problems are done: solved count, score, and a title/rank |

## 4. Behavior Rules

- **Language:** JavaScript submissions only in v1.0.
- **Safe execution:** user code runs in a Web Worker — errors are caught and displayed, and any run longer than the time limit (e.g. 3000 ms) is terminated and reported as `Time Limit Exceeded`. An infinite loop must never freeze the page.
- **Test cases:** every problem ships with visible examples and hidden tests. *Run* uses visible only; *Submit* uses everything.
- **Session flow:** start screen → problems one by one (skipping allowed) → ending screen when the timer hits zero or all problems are answered.

## 5. Problem Format

Each problem is a JSON object:

```json
{
  "id": "two-sum",
  "title": "Two Sum",
  "difficulty": "habit",
  "statement": "Given an array of numbers and a target...",
  "signature": "function solve(nums, target)",
  "visibleTests": [{ "input": [[2, 7, 11], 9], "expected": [0, 1] }],
  "hiddenTests": [{ "input": [[3, 3], 6], "expected": [0, 1] }]
}
```

Difficulty levels use themed names: **gateway** (easy) → **habit** (medium) → **dependency** (hard).

## 6. Constraints

- Single HTML file (HTML + CSS + JS inline); opens directly in a browser.
- No backend, no database, no accounts, no build step.
- No external libraries in v1.0 (a plain `<textarea>` is acceptable as the editor).
- Ships with a small built-in problem set so the site works even without AI generation.

## 7. Out of Scope (v1.0) / Future Ideas

- Syntax highlighting (CodeMirror via CDN)
- Sounds and background music
- Dark mode / themes
- Importing problems via the Codeforces API
- Multiplayer or leaderboards
- Languages other than JavaScript

## 8. Success Criteria

The project succeeds when a player can open the file, play a full timed session of
AI-generated problems, receive correct verdicts for correct/wrong/crashing/looping
code, and see an accurate ending screen — with zero setup.
