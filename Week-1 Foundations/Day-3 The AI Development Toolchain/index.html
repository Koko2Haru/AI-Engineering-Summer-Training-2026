TASK 01 · THE TOOLS, IN MY OWN WORDS

Three tools, one job each

INTERFACE

API

A waiter between my code and someone else's service. I never walk into their kitchen — their servers, their model — I just hand a request to the waiter and it comes back with the answer. The menu is fixed: the API defines exactly what I can ask for and how.

my code → request → their service → JSON → my code

Where it fits: inside the pipeline itself — it's the bridge between my Application and the LLM. Requests go in, answers come back.

TERMINAL

CLI

Talking to the computer with text instead of clicks. One line in the terminal installs a library, runs a script, or pushes code — no hunting through menus. Slower to learn, much faster to use.

pip install requests · git push · curl api.model.com

Where it fits: around the pipeline, not inside it — it's how I install SDKs, run my script, and test API calls while building.

WORKSHOP

IDE

The workshop where the code gets written. Editor, autocomplete, debugger and a terminal, all in one app — so I'm not juggling ten separate tools while I build.

VS Code, PyCharm — mine is VS Code

Where it fits: where the Application step is born. The prompt strings and the API calls are all typed here first.
