# 🐳 Docker Testing Guide

This guide explains how to test the Dockerized Jupyter notebook.

---

## ✅ Prerequisites

Before running the container, make sure:

- Docker Desktop is running.
- Ollama is running on your local machine.
- You are inside the project folder containing:
  - `Dockerfile`
  - `requirements.txt`
  - `prompts_level.ipynb`

---

## 🧹 Step 1: Clear Notebook Outputs

To verify that Docker actually executes the notebook:

1. Open `prompts_level.ipynb`.
2. Click **Clear All Outputs**.
3. Save the notebook (`Ctrl + S`).

This ensures that any outputs you see afterward were generated by Docker.

---

## 🏗️ Step 2: Build the Docker Image

Run the following command from the project directory:

```bash
docker build -t prompt-notebook .
```

> **Note:** You only need to rebuild the image if you change the notebook, the Dockerfile, or the requirements.

---

## ▶️ Step 3: Run the Docker Container

Execute:

```bash
docker run --rm -v "${PWD}:/app" -e OLLAMA_HOST=http://host.docker.internal:11434 prompt-notebook
```

### 📌 What this command does

- `--rm`
  - Removes the container after it finishes.

- `-v "${PWD}:/app"`
  - Mounts the current project folder into the container so any notebook changes are saved back to your computer.

- `-e OLLAMA_HOST=http://host.docker.internal:11434`
  - Allows the Docker container to communicate with the Ollama server running on the host machine.

- `prompt-notebook`
  - The Docker image name.

---

## 🔍 Step 4: Verify the Results

After the container finishes:

1. Open `prompts_level.ipynb`.
2. Confirm that all code cells contain newly generated outputs.

If the outputs are present, Docker successfully executed the notebook.

---

## 🔄 Expected Workflow

```text
Clear Outputs
      │
      ▼
Build Docker Image
      │
      ▼
Run Docker Container
      │
      ▼
Notebook Executes
      │
      ▼
Outputs Saved to Notebook
      │
      ▼
Verify Results
```

---

## 📝 Notes

- Docker Desktop must be running before executing any Docker commands.
- Ollama must be running locally.
- If Docker cannot connect to Ollama, verify that Ollama is running and accessible on `http://host.docker.internal:11434`.
